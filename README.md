# Ishara Project - Indian Sign Language Recognition

This project recognizes Indian Sign Language (ISL) gestures from hand landmarks using MediaPipe + TensorFlow.

## Recommended Project Structure

Use this structure for a clean and portable project setup:

```text
ishara project/
  backend/
    preprocess.py
    train.py
    realtime.py
    model/
      karan.pickle    # generated by preprocess.py
      karan.pkl       # generated by preprocess.py (label encoder)
      karan.h5        # generated by train.py
    .venv/            # keep only ONE virtual environment
  dataset/
    Indian/
      A/ B/ C/ ... Z/
      1/ 2/ 3/ ... 9/
  README.md
  requirements.txt
  .gitignore
```

## What To Keep And What To Delete

Keep:
- `backend/` (all 3 Python scripts)
- `backend/model/` (trained model and encoder files)
- `dataset/Indian/` (training images)
- One virtual environment only (recommended: `backend/.venv`)

Delete:
- `mp_env/` (duplicate environment)
- `myenv/` (duplicate environment)
- Any extra env folder if you keep `backend/.venv`
- `__pycache__/` folders
- C/C++ VS Code files if not needed for this Python project:
  - `.vscode/c_cpp_properties.json`
  - `.vscode/launch.json`

## Essential Environment

- Python: `3.10.x` (your current envs are `3.10.11`)
- Required Python packages:
  - `numpy`
  - `opencv-python`
  - `mediapipe`
  - `scikit-learn`
  - `tensorflow`

## Dataset Format

Your dataset must be organized as:

```text
dataset/Indian/<LABEL>/*.jpg
```

Example labels:
- Alphabet folders: `A` to `Z`
- Number folders: `1` to `9`

## How To Run

Always run from `backend/` folder:

```powershell
cd "C:\ml project\ishara project\backend"
.\.venv\Scripts\Activate.ps1
```

### 1. Preprocess data

```powershell
python preprocess.py
```

Output files (auto-saved in `backend/model/`):
- `karan.pickle`
- `karan.pkl`

### 2. Train model

```powershell
python train.py
```

Output file (auto-saved in `backend/model/`):
- `karan.h5`

### 3. Start real-time prediction

```powershell
python realtime.py
```

Controls:
- Press `q` to quit webcam window.

## Common Issues

- If camera does not open, close apps like Zoom/Teams and run again.
- If `ModuleNotFoundError` appears, verify environment is active and reinstall dependencies.
- If model files are missing, run `preprocess.py` then `train.py` before `realtime.py`.

## Notes

- Scripts are now path-safe and use project-relative directories.
- `realtime.py` reads model files from `backend/model/`.
